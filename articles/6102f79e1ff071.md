## Zod とは

`TypeScript-first schema validation with static type inference`

静的な型推論を持った TypeScript 優先のスキーマバリデーション（構造検証）

以下の特徴をもつ

- 外部依存なし(Zero external dependencies)
- Node.js や あらゆるモダンなブラウザで動作(Works in Node.js and all modern browsers)
- 小さい(コアで2kb程度)(Tiny: 2kb core bundle (gzipped))
- イミュータブルなAPI:メソッドは新しいインスタンスを返す(Immutable API: methods return a new instance)
- 簡潔なインターフェース(Concise interface)
- TypeScript や プレーンな JS で動作(Works with TypeScript and plain JS)
- 組み込みの JSON スキーマ変換(Built-in JSON Schema conversion)
- 広範囲なエコシステム(Extensive ecosystem)

### スキーマの生成

  ```js
  import * as z from 'zod';

  const Player = z.object({
    username: z.string(),
    xp: z.number(),
  });
  ```

### データの解析

- 入力データをバリデートするために、`.parse()` を使う

  ```js
  const res = Player.parse({ username: 'bobchin', xp: 100 });
  console.log(res); // => { username: 'bobchin', xp: 100 }
  ```

- エラーハンドリング
  バリデーションでアンマッチのときは `.parse()` メソッドで `ZodError` をスローする

  ```js
  try {
    Player.parse({ username: 42, xp: "100" });
  } catch(error){
    if(error instanceof z.ZodError){
      error.issues;
      /* [
        {
          expected: 'string',
          code: 'invalid_type',
          path: [ 'username' ],
          message: 'Invalid input: expected string'
        },
        {
          expected: 'number',
          code: 'invalid_type',
          path: [ 'xp' ],
          message: 'Invalid input: expected number'
        }
      ] */
    }
  }
  ```

- `try/catch ` ブロックを使いたくない場合は、`.safeParse()` メソッドを使う

  ```js
  const result = Player.safeParse({ username: 42, xp: "100" });
  if (!result.success) {
    result.error;   // ZodError instance
  } else {
    result.data;    // { username: string; xp: number }
  }
  ```

### 型推論

Zod はスキーマ定義から静的な型を推論する。`z.infer<>()` を使うことができる。

```js
const Player = z.object({
  username: z.string(),
  xp: z.number()
});

// 推論される型を抽出する
type Player = z.infer<typeof Player>;

// コードで使用する
const player: Player = { username: "billie", xp: 100 };
```

input と output のスキーマの型を分けることが可能。`.transform()` API である型から別の型へ変換できる。

```js
const mySchema = z.string().transform((val) => val.length);

type MySchemaIn = z.input<typeof mySchema>;
// 入力 => string

type MySchemaOut = z.output<typeof mySchema>; // equivalent to z.infer<typeof mySchema>
// 出力 => number
```

### プリミティブ型

```js
import * as z from 'zod';

z.string();
z.number();
  z.bigint()
z.boolean();
z.undefined();
z.null();
z.symbol();
```

#### Coercion (型を強制(変換))

```js
z.coerce.string();  // String(input)
z.coerce.number();  // Number(input)
z.coerce.boolean(); // Boolean(input)
z.coerce.bigint();  // BigInt(input)

// example. parse() は変換された値になる
const schema = z.coerce.string();
schema.parse("tuna");    // => "tuna"
schema.parse(42);        // => "42"
schema.parse(true);      // => "true"
schema.parse(null);      // => "null"

// 型強制されたスキーマの入力の型はデフォルトでは unknown になるが、
// ジェネリックパラメータを渡して、型を指定することができる
const A = z.coerce.number();
type AInput = z.input<typeof A>; // => unknown

const B = z.coerce.number<number>();
type BInput = z.input<typeof B>; // => number
```


### リテラル

**リテラルそのもの** を型とする

```js
const tuna   = z.literal("tuna");
const twelve = z.literal(12);
const twobig = z.literal(2n);
const tru    = z.literal(true);

const n = z.null();
const u = z.undefined();
const v = z.void();
```

複数の値も使用可能

```js
const colors = z.literal(["red", "green", "blue"]);

colors.parse("green");  // OK
colors.parse("yellow"); // NG

console.log(colors.values)
```

### Number

```js
const schema = z.number();

schema.parse(3.14);      // OK
schema.parse(NaN);       // NG
schema.parse(Infinity);  // NG
```

数値特定のバリデーション

```js
z.number().gt(5);           // 5より大きい
z.number().gte(5);          // 5以上 = .min(5)
z.number().lt(5);           // 5より小さい
z.number().lte(5);          // 5以下 = .max(5)
z.number().positive();      // 正の数 = .gt(0)
z.number().nonnegative();   // 0以上
z.number().negative();      // 負の数
z.number().nonpositive();   // 0以下
z.number().multipleOf(5);   // 5の倍数 = .step(5)
```

### Nullable

nullのみ許容。undefinedはNG

```js
z.string().nullable().safeParse("abc")      // => true
z.string().nullable().safeParse(null)       // => true
z.string().nullable().safeParse(undefined)  // => false
```

### Nullish

?? 演算子と同じか。nullもundefinedも許容する

```js
z.string().nullish().safeParse("abc")     // => true
z.string().nullish().safeParse(null)      // => true
z.string().nullish().safeParse(undefined) // => true
```
