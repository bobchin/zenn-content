## バリデーション

[Hono だけでバリデーション](#hono-だけでバリデーション) する方法と [Zod を使ってバリデーション](#zod-を使ってバリデーション) する方法がある。
`Hono だけでバリデーション` が基本の構成で、実際は Zod を使うのがよいように思う。
また Zod を使う場合、zod-validator を使用すると記述が簡潔になる。

### Hono だけでバリデーション

`hono/validator` を使う。
基本的にはミドルウェアで、個別のルーティングに指定してハンドラの前に実行している。
バリデーションの対象指定のキー名("form" など)のオブジェクトに結果を格納して、
後で実行されるハンドラで取得できるようにしている。
ハンドラでは、c.req.valid('form') で結果を受け取って処理をすることができる。

- バリデータ
  - 第1引数: バリデーションの対象(json, query, header, param, cookie も指定可能)
    - form    : c.req.bodyCache.formData
    - json    : c.req.json()
    - query   : c.req.queries()
    - header  : c.req.header()
    - param   : c.req.param()
    - cookie  : c.req.raw.headers.get('Cookie')

  - 第2引数: バリデーションの処理
    - コールバックの第1引数: バリデーション対象の値
    - コールバックの第2引数: Context

- ハンドラ
  - ハンドラー内では、c.req.valid('form') でバリデーション処理で返した値を取得できる
  - c.req.valid() の引数は、第1引数で指定したバリデーションの対象名

```js
import { validator } from 'hono/validator'
import { Hono } from 'hono'

const app = new Hono()

app.post(
  '/posts',
  // バリデータ
  validator(
    'form',
    (value, c) => {
      const body = value['body']

      if (!body || typeof body !== 'string') {
        return c.text('Invalid!', 400)
      } else {
        return { body: body }
      }
    }
  ),
  // ハンドラ
  (c) => {
    const { body } = c.req.valid('form')
    return c.json({ message: 'Created!' },　201)
  }
)
```

### Zod を使ってバリデーション

- インストール

  ```bash
  npm install zod
  npm install @hono/zod-validator
  ```

- Zod を使用してバリデーションする

  - スキーマを定義する
  - safeParse() でバリデーションする

  ※[Zod](https://zenn.dev/bobchin/articles/6102f79e1ff071) の使い方はリンクを参照

https://zenn.dev/bobchin/articles/6102f79e1ff071


  ```js
  import z from 'zod'
  import { Hono } from 'hono'

  const app = new Hono()

  // Zod でスキーマの定義
  const schema = z.object({
    body: z.string(),
  })

  app.post(
    '/posts',
    // バリデータ
    validator(
      'form',
      (value, c) => {
        const parsed = schema.safeParse(value)
        if (!parsed.success) {
          return c.text('Invalid!', 401)
        } else {
          return parsed.data
        }
      }
    ),
    // ハンドラ
    (c) => {
      const { body } = c.req.valid('form')
      return c.json({ message: 'Created!' },　201)
    }
  )
  ```


- zod-validator を使うと上記処理を簡素化できる

  ```js
  import z from 'zod'
  import { zValidator } from '@hono/zod-validator'
  import { Hono } from 'hono'

  const app = new Hono()

  // Zod でスキーマの定義
  const schema = z.object({
    body: z.string(),
  })

  app.post(
    '/posts',
    // バリデータ
    zValidator('form', schema),
    // ハンドラ
    (c) => {
      const { body } = c.req.valid('form')
      return c.json({ message: 'Created!' },　201)
    }
  )
  ```

    - zValidator の中身のイメージ

      ```js
      function zValidator(target, schema) {
        return async (value, c) => {
          const result = await schema.safeParseAsync(value)
          return (result.success)? result.data: c.json(result, 400)
        }
      }
      ```
