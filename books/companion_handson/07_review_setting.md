---
title: "ボタンの設定を確認する"
---

## ボタンの設定を確認してみる

左側に追加した `In1` をクリックする。

## ローカル変数を確認する

ローカル変数とは、**`ボタン自体に紐つく変数`** です。
内容を確認してみるため、右側の `Local Variables` をクリックします。

以下の変数があることを確認します。

- `internal: User Value`
  - Valiable name: me
  - Current Value: 1
  - [x] Persist value

- `internal: User Value`
  - Valiable name: input
  - Current Value: 1
  - [x] Persist value

:::message
- Valiable name: 変数名です
- Current Value: 変数の値です
- Persist value: チェックを入れると、変数ではなく `定数` になります。
:::

:::message
In1 ～ In4 は、入力番号以外に違いがないため、ボタン自体に番号だけを変数で可変にしています。
:::

:::message
変数には2種類あり、`ローカル変数` と `グローバル変数` があります。
- `ローカル変数` が使用できる場所では、`「地球」アイコン` が表示されます
- `グローバル変数` が使用できる場所では、`「＄」アイコン` が表示されます
:::

## アクションを確認する

アクションとは、**`ボタンを押した(Press actions)ときの動作`** です。
内容を確認してみるため、右側の `Step1` をクリックします。

Press actions には、プリセットアクションの `atem: ME: Set Program input` がセットされています。
値として、ローカル変数 `$(local:input)` を参照しています。ここでは値が `1` となります。
変数の実際の値は、`Input` の下に青い枠で表示されます。

ボタンを押すと、プリセット機能で準備されている `Set Program input` をコールし、値に `1` をセットすることで、プログラム入力の１が選択されます。

## スタイルを確認する

スタイルとは、**`ボタンの見た目`** です。
内容を確認してみるため、右側の `Style` をクリックします。

- Text をクリックします。`Button text string` でボタンに表示する文字列を指定できます。
- Image をクリックします。`Image` でボタンに表示する画像を指定できます。
- Background をクリックします。`Color` でボタンの背景色を指定できます。

:::message
Text では、変数 `$(atem:short_$(local:input))` が指定されています。
`$(変数名)` で変数で値に展開されます。
ローカル変数 `$(local:input)` の値が `"1"` のため、`$(atem:short_1)` に展開されます。
Connection 変数 `"atem:short_1"` は、atem コネクションで用意されている変数です。
Atem で設定している入力のショートネーム `"In 1"` に展開されます。
:::

## フィードバックを確認する

フィードバックとは、**`変数の値などの条件による見た目の指定`** です。
内容を確認してみるため、右側の `Feedbacks` をクリックします。

Feedbacks には、プリセットフィードバックの `atem: ME: One ME program source` がセットされています。
値として、ローカル変数 `$(local:input)` を参照しています。ここでは値が `1` となります。
変数の実際の値は、`Input` の下に青い枠で表示されます。

Atem のプログラムソースが `1` のときに、ここで設定した見た目が適用されます。

Layered Styles Overrides では、
Text: 白色
Background: 赤色
が設定されているため、プログラムソースの１を選択した場合に、背景が赤で白文字で表示します。

:::message alert
注意
Atem 実機の状態を参照しているため、ネットワーク経由で Atem にアクセスしていない場合は正常に動作しません。
:::
